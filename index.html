Förberedelser

Workshopen kommer att bestå till stor del av praktiska övningar. Det är därför viktigt att alla som vill delta i workshopen ska förbereda sina utvecklingsmiljöer enligt följande instruktioner:

  • Installera Scala
  • Installera sbt
  • Installera Eclipse eller Intellij
  • Installera Scala IDE för Eclipse eller Scala plugin för Intellij
  • Hämta filen build.sbt och lägg den på sökvägen <hemkatalogen>/.sbt/plugins/ (skapa katalogen om den inte redan finns)
  • Installera Git

Om man vill kunna köra sbt i Ecllipse eller Intellij så kan man:

  • Installera idea-sbt plugin för Intellij
  • Följa instruktionerna för att köra sbt i terminalfönstret i Eclipse

Övningar

Lösningarna på följande övningar kan man ladda ner och öppna i IDE:n, genom att:

  • I terminalfönstret, skriv: git clone git://github.com/uzilan/scalaworkshop.git
  • Skriv cd scalaworkshop
  • Om du använder Eclipse, skriv sbt eclipse. För Intellij, skriv sbt gen-idea
  • Öppna den nerladdade projektet i Eclipse eller Intellij

Scala REPL

  • Använd REPL:en för att skapa olika variabler (både automatiska och namngivna) och kombinera dem med varandra
  • När kan man använda autocomplete i REPL:en?
  • Skapa ett "Hello World" meddelande mha två variabler
  • Givet att val tjo = "tjohej", vad händer om man skriver tjo * 10?

Classes & objects

  • Skapa en class Fraction med attribut num och denom i konstruktorn.
  • Överlagra metoden toString i klassen
  • Skapa ett object Fraction i samma fil

Case classes

  • Skapa en case class Person med några lämpliga attribut i konstruktorn och använd sedan javap för att inspektera vilka metoder den har
  • Hur många rader kod skulle det krävas för att skriva motsvarande klass i Java? :)

Scaladoc

  • När man skapar en List i Scala får man en förvald immutable lista. Vilket packet finns den definerad i? Var kan man hitta en mutable lista?
  • I scaladoc kan man hitta klasserna Tuple1 och Tuple2. Finns det fler Tuples? Hur många?
  • Varför finns inte klassen String i Scaladoc? Var finns alla metoder i String?

Traits & Mixins

  • Skapa en trait Touchscreen med ett attribute touchscreen: String med ett lämpligt värde
  • Skapa en trait Apple med ett attribute apple: String med ett lämpligt värde
  • Skapa en abstract class Thing med ett attribute stuff: List[String] och en metod som skriver ut innehållet i listan (använd List.mkString)
  • Skapa en class Android som ärver från Thing och mixar Touchscreen och som lägger till touchscreen i stuff
  • Skapa en class IPhone som ärver från Thing och mixar Touchscreen och Apple och som lägger till touchscreen och apple i stuff
  • Skapa ett object Things som ärver från App, och som skriver ut innehållet i en Android och en IPhone

Functions

  • Skapa en case class User med för- och efternamn som argument
  • Skapa ett object User med följande hjälpfunktioner:
    • doWithUser tar en User och en funktion som tar User och returnerar en String och appliceras på Usern. Funktionen ska returnera en String
    • fullName tar en User och returnerar hans fullständiga namn
    • userFullName tar en User och anropar doWithUser med Usern och funktionen fullName som argument
    • scrambleName tar en User och returnerar hans namn med blandade bokstäver. (Titta på scala.util.Random för lämplig funktion)
  • Skapa ett program som:
    • importerar metoderna i objektet User mha koden import User._
    • skapar en User kalle
    • anropar och skriver ut resultatet av userFullName med kalle
    • anropar och skriver ut resultatet av doWithPerson med kalle och scrambleName som argument

sbt

  • I terminalfönstret, skapa en katalog "sbttest"
  • byt till katalogen genom att köra cd sbttest
  • starta sbt REPL genom att köra sbt
  • skriv set name := "sbttest"
  • skriv set organization := "diversify.se"
  • skriv session save
  • skriv eclipse with-source=true (även om du tänker köra intellij, annars skapas inte katalogstrukturen)
  • skriv gen-idea (bara om du tänker köra intellij)
  • skriv test
  • öppna projektet i eclipse/intellij

Voila! nu har du skapat ett sbt projekt med katalogstruktur, färdig att öppnas som projekt i eclipse eller intellij.

Test

  • Skapa ett sbt projekt
  • Lägg till support för specs2 och junit genom att skriva följande efter en tom rad (viktigt!) i build.sbt:

libraryDependencies ++= Seq( "org.specs2" %% "specs2" % "1.7.1" % "test", "junit" % "junit" % "4.8.2" % "test" )

  • Skapa en trait Shape med metoder area: Double och perimeter: Double
  • Skapa case class Square och case class Circle som implementerar Shape och räknar ut sina area respektive omkrets
  • Skriv tester för klasserna Square och Circle

Trollcount

Den sista övningen heter Trollcount och är lite mer complex. Övningen kan göras på olika sätt, beroende på hur säker man känner sig på Scala. Gör följande för att börja med övningen:

  • Läs om Trollcount på http://codebyexample.info/2012/04/24/trollcount-count-like-a-troll/
  • Hämta övningen: git clone git://github.com/uzilan/trollcount.git
  • Kör cd trollcount
  • Välj vilken branch man vill jobba i genom att köra git checkout <branchnamn>:
    • tests-only - innehåller endast tester - Välj den branchen om du vill implementera hela logiken själv.
    • tests-and-stubs - innehåller tester och stubbar - välj den branchen om du vill ha hjälp med att börja implementera lösningen.
    • master - den fullständiga lösningen. Välj den branchen om du vill gå igenom min lösning.
  • Kör sbt ~test för att börja testa applikationen
  • Implementera!

Övningen går på att få lösningen att kompilera och testerna att vara gröna.
