Preparations

This workshop mainly consists of practical exercises. It is therefore important that all it's attendees will prepare their development environments according to the following instructions:

  • Install Scala
  • Install sbt
  • Install Eclipse eller Intellij
  • Install Scala IDE for Eclipse or Scala plugin for Intellij
  • Fetch the build.sbt file and add it in your <home>/.sbt/0.13/plugins/ path (create the folder if it is missing)
  • Install Git

If you want to run sbt inside Ecllipse or Intellij:

  • Install idea-sbt plugin for Intellij
  • Follow the instruktions in run sbt inside eclipse i Eclipse

Q&A

What do I need to do to be able to run scalaand scalac?

  • Add your scala/bin folder to your PATH environment variable

How do I install sbt?

  • Follow the sbt installation instructions

How do I create a Scala project in Eclipse?

  • Choose File - New - Other - Scala Wizards - Scala Project

How to create a Scala project in Intellij?

  • Choose File - New Project - Create project from scrach
  • Choose Next
  • Choose Java Module and add a name etc.
  • Choose Next
  • Choose Create source directory
  • Choose Next
  • Click in the Scala checkbox. Under Use Scala Distribution, choose your Scala installation folder
  • Choose Finish

How do I run javap?

  • In the terminal, go to the folder containing the file you want to investigate.
  • Compile it using the scalaccommand, e.g. scalac animals.scala
  • run javapon the class you compiled, e.g. javap Cat

Exercises

Solutions to the following exercises can be downloaded and opened in your IDE using the following instructions:

  • In the terminal, write: git clone git://github.com/uzilan/scalaworkshop.git
  • write cd scalaworkshop
  • If you use Eclipse, write sbt eclipse. For Intellij, write sbt gen-idea
  • Open the newly created project in your IDE.

Scala REPL

  • Use the REPL to create different variables (both anonymous and named) and combine them with each other
  • When can one use autocomplete in the REPL?
  • Create a "Hello World" message using two variables
  • Given that val foo = "bar", what would happen if you run foo * 10?

Classes & Objects

  • Create a class Fraction with attributes num and denom in the constructor
  • Override method toString inside the class
  • Create an object Fraction in the same file
  • Create an Apply factory method inside the object, which creates instances of the Fraction class.
  • Create an implicit converter which converts Int to Fraction instances inside the object
  • Create an object Fractions which inherits from App, which tests creating Fractions with and without new, as well as converting Ints into Fractions

Case Classes

  • Create a case class Person using adequate attributes in the constructor.
  • Use the javap command to inspect which methods it contains
  • How much code would you expect a similar Java implementation would require? :)

Scaladoc

  • When creating a List in Scala, the default implementation is an immutable list. In which package is it defined? Where can one find a mutable list?
  • In Scaladoc there are definitions for classes Tuple1 and Tuple2. Are there more Tuples? How many?
  • Why can't we find the String class definition in Scaladoc? Where can we find the methods in String?

Traits & Mixins

  • Create a trait Touchscreen with attribute touchscreen: String with value "a touchscreen"
  • Create a trait Apple with attribute apple: String with value "an Apple"
  • Create an abstract class Thing with an attribute stuff: List[String] and a method that writes the contents of the list (use the List.mkString method)
  • Create a class Android that inherits from Thing and mixes Touchscreen, and adds touchscreen to stuff
  • Create a class IPhone that inherits from Thing and mixes Touchscreen and Apple, and adds touchscreen and apple to stuff
  • Create an object things that inherits from App, which prints the contents in an Android and an IPhone instances

Functions

  • Create a case class User with given name and a sirname as arguments
  • Create an object User with the following help functions:
    • doWithUser takes a User and a function that takes a User as argument and returns a String. The function should return a String
    • fullName takes a User and returns it's full name
    • userFullName takes a User as argument and calls doWithUser with the User and the fullName function and returns the result
    • scrambleName takes a User as argument and returns it's name with mixed letters. (Take a look at scala.util.Random)
  • Create a program that:
    • Imports the methods in object User using import User._
    • Creates a User Bertil
    • Calls and prints the result of calling userFullName with Bertil as argument
    • Calls and prints the result of calling doWithPerson with Bertil and scrambleName as arguments

Loops

  • Create a program that prints the multiplication table
  • Create a for loop that takes a list with weekdays in short format (Sun, Mon, Tues, ...) and creates a list with complete weekday names (Sunday, Monday, Tuesday, ...) using for and yield
  • Using a for loop, print every third word in the string: "I have never thought I'm gonna need to give blood again. You saw me up there, right?"

Recursion

  • Create a program which prints the names of the files in a given folder, recursively
  • Create a program that reverses a list recursively

sbt

  • In the terminal, create a folder named "sbttest"
  • Change directory to the folder by running cd sbttest
  • Start the sbt REPL by running sbt
  • Write set name := "sbttest"
  • Write set organization := "webstep.no"
  • Write session save
  • Write eclipse with-source=true (even if you are running intellij, otherwise no folder structure will be created)
  • Write gen-idea (only if you run intellij)
  • Write test
  • Open the project in your IDE

Voila! You have now created an sbt project with folder structure which can be opened in your IDE.

Test

  • Create a new sbt project
  • Add support for specs2 and junit by adding the following after an empty row (note: important! After an empty row!) in your build.sbt file.
libraryDependencies ++= Seq(
  "org.specs2" %% "specs2" % "2.3.3" % "test",
  "junit" % "junit" % "4.8.2" % "test"
)
  • Create a trait Shape with method area: Double and perimeter: Double
  • Create a case class Square and a case class Circle which implements Shape and it's methods
  • Create tests for classes Square and Circle

Trollcount

The last exercise is a bit more complex. The exercise can be completed in various ways, depending on how well you feel you understand Scala. Follow these instructions to begin:

  • Read about Trollcount at http://codebyexample.info/2012/04/24/trollcount-count-like-a-troll/
  • Fetch the exercise: git clone git://github.com/uzilan/trollcount.git
  • Choose which branch to work with by writing git checkout <branchnamn>:
    • tests-only - only contains tests and no implementations. Choose this branch if you want to implement the program yourself.
    • tests-and-stubs - contains tests and stubs. Choose this branch if you need help beginning the implementation.
    • master - the complete solution. Choose this branch if you want go through the solution.
  • Run sbt ~test to begin testing the implementation
  • Implement!

The exercise's goal is to see to that all tests become green.
